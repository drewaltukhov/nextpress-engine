NextPress is a modern publishing engine for people who want a real CMS without the legacy weight. If you've used WordPress, you already know how it thinks — posts, topics, plugins, themes — except every piece is type-safe, server-first, and built on Next.js.

> **New here?** Jump straight to [Getting started](/docs/getting-started) — installation, the first-run wizard, and your first post in under ten minutes.

## What carries over from WordPress

The mental model. If you've built sites on WordPress, NextPress will feel familiar from day one.

| WordPress | NextPress |
| --- | --- |
| Posts & post types | Posts with typed schemas and revisions |
| Pages | Pages with the same block-based editor as posts |
| Categories & tags | Topics (flat) + Pillars & spikes (hub-and-spoke) |
| Actions & filters | Type-safe hook bus (actions + filters) |
| Plugins | Isolated plugins with their own migrations, settings, and routes |
| Themes | Swappable presentation layer with a visual builder |
| Options table | Settings registry with Zod validation and encryption |
| Roles & capabilities | Roles with granular permissions + step-up auth |
| Sitemap / RSS / robots.txt | Built-in and load-bearing — they ship in the essential tier |

## What doesn't

- **No PHP**, no themes-from-the-marketplace roulette, no "incompatible with WordPress X.Y" Tuesdays.
- **No tracking.** The engine doesn't ship analytics, tracking pixels, or third-party data collection.
- **No plugin marketplace overhead.** Plugins are TypeScript files in your repo. Install one by `git clone`-ing it; uninstall by deleting it.

## Core principles

- **Plugin-first architecture.** Posts, users, media, auth — every feature flows through the same plugin contract.
- **SEO survives everything.** Sitemaps, structured data, RSS, robots.txt are essential-tier and load even when other plugins fail.
- **Security is the baseline.** Lockout, IP blocking, audit logging, encrypted credentials, step-up auth — all in core.
- **Privacy by default.** No tracking pixels, no third-party analytics, no data you didn't ask for.

## Two-minute tour

1. **You install NextPress** as a Next.js app (local, Vercel, or self-hosted).
2. **You sign in** and finish the first-run wizard — site title, admin user, timezone.
3. **You write content** in the admin UI — pages, posts, topics, media.
4. **The active theme renders** your content on the public site. The default `nextpresso` theme covers most use cases out of the box.
5. **You customize**: theme builder for visual changes, settings for behavior changes, the REST API for headless consumers.
