These guides walk you through every screen in the admin you'll use day-to-day. They assume you've finished the [first-run wizard](/docs/getting-started) and signed in.

Each guide is self-contained — you can jump straight to the one you need without reading the others.

## Content you publish

| Guide | What it covers |
| --- | --- |
| [Pages](/docs/guides/pages) | One-off static pages: About, Contact, Privacy, landing pages. URL lives at the root (`/about`). |
| [Posts](/docs/guides/posts) | Articles, blog entries — anything with a published-at date. URL is the post slug. |
| [Topics](/docs/guides/topics) | Taxonomies. Tag posts with topics so visitors can browse `/topics/<slug>` archives. |
| [Pillars & spikes](/docs/guides/pillars-and-spikes) | Two post kinds for "hub-and-spoke" content. A *pillar* is a long-form overview; *spikes* are focused articles that hang off it. |
| [Media library](/docs/guides/media) | Upload, organize, and embed images, video, and other files. |

## Site structure

| Guide | What it covers |
| --- | --- |
| [Menus](/docs/guides/menus) | Build the navigation bar, footer menus, anywhere a list of links shows up. |
| [Mega menus](/docs/guides/mega-menu) | Big drop-down panels under top-level items — featured posts, link columns, image tiles, rich text. |
| [Users & authors](/docs/guides/users) | Invite editors, assign roles, manage author bios. |
