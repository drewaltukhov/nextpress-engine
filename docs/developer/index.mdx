## How NextPress thinks about extensibility

Every feature in NextPress — users, posts, media, security, even the SEO meta tags — is a plugin. Core plugins live under `src/core-plugins/`; third-party plugins live under `src/plugins/`. The contract is the same. There's no "WordPress vs plugins" two-tier model — if you can do it in core, you can do it in a plugin.

The full plugin contract lives in [Plugin system](/docs/developer/plugins). The REST API surface — for headless consumers, mobile clients, integrations — is in [API reference](/docs/developer/api).
